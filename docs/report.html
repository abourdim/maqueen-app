🛠 CI & quality

Every push and PR to main runs two CI jobs. They block merge if anything regresses.

JobChecksFailure mode
audit ✅ Link audit (53 HTML · 0 broken hrefs · 1 allowed orphan)
✅ Inlined-md staleness — refuses commits that edited .md without re-running the inliner
✅ JSON validity — product.json + build-info.json
✅ Secret-leak grep — API_KEY= · PRIVATE_KEY · TODO:.*SECRET
✅ A11y sanity — every <html> has lang, every <img> has alt 		Hard-fails on broken hrefs or stale inlined md
smoke ✅ Playwright headless (chromium)
✅ 32 tests covering hubs · labs · lessons · auto-render · demo-mode · telemetry · share-link · a11y · lang switcher · workshops hub flyer/poster
✅ Total runtime ~1.3 min in CI 		Fails on any test failure; uploads Playwright report on failure

Local quality tools

  • npm test — run smoke tests
  • node docs/_inline-md.mjs — refresh inlined markdown blocks
  • python tools/_audit-links.py --strict — link audit (CI mode)
  • python tools/_check-asset-refs.py path/to/x.html — verify asset refs exist
  • node etsy-package/build-package.js — build the buyer ZIP

🔇 Explicitly won't do

  • Translate technical docs (CHANGELOG.md, CUSTOMIZE.md) — contributor-facing, EN is fine
  • Cross-browser smoke tests (Firefox/WebKit) — Web Bluetooth is Chromium-only anyway
  • Virtual physics layer for demo-mode — current synthetic responses are enough; full physics would 5-10× the code
  • Modify js/ble.js — stable forked dependency from bit-playground; rule is to wrap, never patch
