# micro:bit Playground — دليل المستخدم

> **ملاحظة:** هذا الدليل مكتوب بالعربيّة (اتجاه النصّ من اليمين إلى اليسار). يعرض Markdown العاديّ المحتوى من اليسار إلى اليمين افتراضيًّا، لكنّ النصّ ذاته عربيّ فصيح.

> **micro:bit Playground** — لوحة تحكّم BLE عبر المتصفح للـ BBC micro:bit V2.
> دون تثبيت، دون حساب، دون خادم خلفيّ. كلّ شيء يعمل محلّيًّا في Chrome أو Edge.

---

## بدء سريع (5 دقائق)

1. ابرمج `makecode.ts` إلى **BBC micro:bit V2** من [makecode.microbit.org](https://makecode.microbit.org/) (الصق في وضع JavaScript ← تنزيل ← اسحب `.hex` إلى قرص MICROBIT).
2. حين تُظهر اللوحة أيقونة **X**، فهي تبثّ BLE.
3. افتح `index.html` في **Chrome** أو **Edge** (سطح المكتب أو Chrome على Android).
4. انقر **🔗 الاتصال بـ micro:bit**، واختر لوحتك من المُنتقي.
5. تتحوّل شارة الحالة إلى **الأخضر** ← أنت متّصل. جرّب الألسنة الثمانية.

الشرح المفصّل: [start-ar.html](start-ar.html). الأسئلة الشائعة: [faq-ar.html](faq-ar.html).

## المتطلّبات

- **BBC micro:bit V2** (V1 يعمل لكن بلا مستشعر صوت).
- كابل USB للبرمجة، بطاريّات أو USB للتشغيل.
- **Chrome** أو **Edge** على سطح المكتب · **Chrome** على Android.
- تفعيل Bluetooth 4.0+ (BLE) على حاسوبك.
- HTTPS أو `file://` — النقر المزدوج على HTML يعمل.

## الألسنة الثمانية

### 1 · 🎛 التحكّم
- **مُمرِّر النصوص** — اكتب رسالة، تمرّر على شاشة LED.
- **مصفوفة LED** — انقر/اسحب لرسم نمط 5×5، يُرسَل بالسادس عشر (`LM:1F0E040000`).
- **إعدادات جاهزة** — قلب ❤، ابتسامة 😊، علامة صحّ ✔ (نقرة واحدة).
- **الأوامر** — HEART / SMILE / SAD / CLEAR / FIRE / الأسهم.
- **الجرس** — منزلق الترّدد (20–20 000 هرتز)، المدّة، 4 إعدادات مسبقة.
- **JSON مخصّص** (خبير فقط) — إرسال حمولات خام `JSON:{…}`.

### 2 · 👀 المستشعرات
قيم مباشرة + مخطّطات مصغّرة، تتحدّث كلّ 100–200 مِلّي ثانية:
- 🌡 الحرارة (°م)
- 💡 الضوء (0–255)
- 🔊 الصوت (V2 فقط، 0–255)
- 🏃 مقياس التسارع X / Y / Z (mg)
- 🧭 اتّجاه البوصلة (0–360°)
- 🔘 الزرّان A و B
- ✋ اللمس P0 / P1 / P2 + الشعار

**لوحة المعايرة** (كلّها يُشغّلها المستخدم، وتُحفَظ في localStorage):

| المعايرة | الطريقة |
|---|---|
| 🧭 البوصلة | انقر معايرة ← أمِل اللوحة لملء الشاشة |
| ⚖️ تصفير مقياس التسارع | ضع مسطّحة ← تحديد المستوى |
| 🔊 قاعدة الصوت | غرفة هادئة ← تحديد المحيط |
| 💡 قاعدة الضوء | إضاءة اعتياديّة ← تحديد المحيط |
| 🔧 ضبط السيرفو (خبير) | لسان المحركات — منزلق −15° إلى +15° لكلّ سيرفو |

### 3 · ⚙️ المحركات
- **السيرفو 1** على P1 · **السيرفو 2** على P2.
- منزلق 0°–180° + مُدخَل رقميّ + مقياس مرئيّ.
- يُحرّر زرّ **OFF** إشارة PWM ويُعيد تفعيل اللمس على ذلك الطرف.
- إرسال زاوية سيرفو يُعطّل تلقائيًّا استطلاع اللمس على طرفه للأمان.

### 4 · 🎮 GamePad
- اتجاهات ⬆⬇⬅➡ + زرّ 🔥 FIRE.
- يُرسل `CMD:UP / DOWN / LEFT / RIGHT / FIRE` على UART.
- رائع لقيادة الروبوتات أو ألعاب LED.

### 5 · 📈 الرسم البياني
- **5 أنواع رسوم** — خط · أعمدة · مبعثر · منطقة · زمن حقيقيّ (راسم ذبذبات).
- **مربّعات اختيار المستشعرات** — Accel X/Y/Z، البوصلة، الصوت، الضوء، الحرارة، اللمس P0/P1/P2.
- **تسميات مخصّصة** — أيّ `GRAPH:<label>:<value>` يرسله البرنامج الثابت يُنشئ خطًّا ملوّنًا آليًّا.
- **10 ألوان صديقة لعمى الألوان**.
- **أدوات التحكّم** — حجم النافذة · محور Y · سماكة الخطّ · الشبكة.
- **الإجراءات** — محاكاة · إيقاف مؤقت · مسح · ملء الشاشة · تسجيل · إعادة تشغيل · حفظ · ملاحظة · PNG · CSV.

### 6 · 🎲 اللوحة ثلاثيّة الأبعاد
نماذج Three.js تعمل ببيانات المستشعرات المباشرة. اسحب للتدوير، مرّر للتكبير:

| النموذج | يتحرّك بـ |
|---|---|
| 🎲 micro:bit V2 | LEDs، الإمالة، الأزرار، الأطراف، الشعار، تدرّج الحرارة |
| 🚗 Robot Buggy | السيرفو 1 (التوجيه)، Accel، LEDs، الضوء، الزرّ A |
| 🦾 ذراع روبوت | السيرفو 1 (القاعدة)، السيرفو 2 (الرفع)، الزرّان A/B (القبضة) |
| 🎯 لعبة التوازن | Accel X/Y (محاكاة فيزيائيّة، كرة على منصّة) |
| 🌦 محطة الأرصاد | الحرارة، الضوء، الصوت، البوصلة |

عناصر التحكّم: مُنتقي النموذج · إعادة ضبط العرض · تدوير تلقائيّ · Live Sync.

### 7 · 🔧 Bench (خبير)
- أرسِل أوامر خام `BENCH:PING / STATUS / RESET`.
- ردود البرنامج الثابت مرئيّة مباشرة.
- مساحة للتجريب وتصحيح الأخطاء.

### 8 · ✨ المزيد
**أدوات ترفيهيّة** (مرئيّة دائمًا): زرّ · مفتاح · منزلق · لوحة أرقام · عصا تحكّم.
**للخبراء فقط**: مؤشّر LED · شريط مستوى · رسم حيّ · متعدّد الرسوم · وحدة تصحيح · التقاط بيانات · جرس · مؤقّت · إجراء مؤجَّل · عشوائيّ · مُنتقي وضع · مُدخَل رقميّ · نطاق مزدوج · مُنتقي لون · إعدادات مسبقة · إعادة تصفير عامّة · سمة · لوحة XY · مصفوفة LED · محاكاة مستشعرات · تحكّم بالأطراف · سيرفو · شريط RGB · إدخال/إخراج ملفّات.

## مرجع سريع لبروتوكول BLE

### الوارد (micro:bit ← المتصفّح)
```
TEMP:23
LIGHT:142
SOUND:87
ACC:120,-45,-980
COMPASS:274
BTN:A:1
BTN:LOGO:0
LEDS:10,31,31,14,4
GRAPH:Distance:42
```

### الصادر (المتصفّح ← micro:bit)
```
TEXT:Hello!
LM:1F0E040000
CMD:HEART
SERVO1:90
SERVO1:OFF
BUZZ:440,200
CAL:COMPASS
SIMULATE:ON
```

المرجع الكامل في `README.md` الخاصّ بالمشروع.

### تفاصيل الاتصال
- معرّف الخدمة: `6e400001-b5a3-f393-e0a9-e50e24dcca9e` (Nordic UART)
- حمولة MTU: 20 بايت (تجزئة تلقائيّة)
- إعادة الاتصال التلقائيّة: 3 محاولات بفارق ثانيتين
- فلتر الجهاز: اسم يبدأ بـ `BBC micro:bit`

## استكشاف الأخطاء

### زرّ "الاتصال" لا يفعل شيئًا
أنت على Safari أو Firefox أو iPhone. انتقل إلى Chrome أو Edge (سطح المكتب أو Chrome على Android).

### لا يظهر micro:bit في نافذة الإقران
1. هل تُظهر اللوحة **X**؟ إن لا، أعد برمجة `makecode.ts`.
2. هل Bluetooth مُفعَّل على مستوى النظام (لا المتصفّح فقط)؟
3. هل دونغل Bluetooth لديك 4.0+ (BLE)؟ الدونغلات القديمة لا تدعم سوى Classic.
4. أغلق `chrome://bluetooth-internals` إن كان مفتوحًا — قد يحتجز المسح.

### تظهر "أمِل لملء الشاشة" عند الإقلاع
برنامج ثابت قديم. أعد برمجة `makecode.ts` الحاليّ من تنزيلك. البرنامج الجديد لا يُشغّل معايرة البوصلة إلّا حين تنقر **معايرة** في لسان المستشعرات.

### قراءة مستشعر الصوت 0
لديك V1. ترقَّ إلى V2 أو تقبّل غياب الصوت (كلّ شيء آخر يعمل).

### انقطاع بعد ثوانٍ
اقترب، غيّر البطاريّات، أو أعد البرمجة. يُعيد التطبيق الاتصال آليًّا 3 مرّات.

### السيرفو يُفسد قراءات اللمس
متوقّع — PWM على الطرف يُفسِد اللمس. انقر **OFF** بجانب السيرفو لتحرير الطرف.

### النموذج ثلاثيّ الأبعاد متلعثم
أغلق الألسنة الأخرى. عطّل التدوير التلقائيّ. جرّب نموذج micro:bit V2 الخفيف. أوقف Live Sync لحظةً.

### الرسم بملء الشاشة لا يُغلَق
اضغط **Esc**، أو انقر أيقونة ⛶ مجدّدًا.

## ملاحظات للمعلّمين

### تسلسل ورشة (45 دقيقة)
1. **0–10 دقائق** — الجميع يُبرمج البرنامج الثابت من MakeCode. أظهِر X على شاشة LED.
2. **10–20 دقيقة** — الإقران عبر BLE. جرّب اللسان 1 (التحكّم) واللسان 2 (المستشعرات).
3. **20–35 دقيقة** — اختر موضوع مشروع (buggy، محطة أرصاد، لعبة توازن). استعمل اللسان 6 (3D) للتخيّل.
4. **35–45 دقيقة** — سجّل جلسة رسم لمدّة دقيقة (اللسان 5). صدِّر CSV لدرس متابعة.

### نصائح الصفّ
- استعمل سمتَي **Arctic** أو **Blaze** لوضوح العرض.
- micro:bit واحد + حاسوب واحد لكلّ ثنائيّ. الإقران BLE لكلّ لسان متصفّح.
- عطّل التدوير التلقائيّ في اللسان 6 حتى لا يُشتَّت التلاميذ.
- زرّ **محاكاة** في اللسان 5 يولّد بيانات تجريبيّة دون مستشعرات — مفيد عند تشارك العتاد.
- فعّل وضع **الخبير** للتلاميذ الأكبر سنًّا فقط — يُخفي فخّ JSON الخام عن المبتدئين.

### أفكار دروس
- 🚗 **Robot buggy** — GamePad + السيرفوات لقيادة روبوت، ومشاهدة Buggy ثلاثيّ الأبعاد يحاكيه.
- 🦾 **ذراع روبوت** — التحكّم بسيرفوين، ورؤية الذراع ثلاثيّ الأبعاد يستجيب في الزمن الحقيقيّ.
- 🌡️ **محطة أرصاد** — رسم الحرارة والضوء، وتحريك محطة الأرصاد ثلاثيّة الأبعاد.
- 🎯 **لعبة توازن** — إمالة اللوحة لتدحرج كرة نحو الأهداف.
- 🎵 **عدّاد صوت** — مراقبة الضوضاء مع شروحات.
- 📊 **صحافة البيانات** — تسجيل جلسة، وتصدير PNG + CSV.

### ملاحظات أمان
- اعتبر سيرفوات micro:bit عند 5V — يُنصَح بطاقة خارجيّة للأحمال عالية العزم.
- الجرس مرتفع — ابدأ بنسبة 50٪ من الصوت.
- مدى BLE نحو 10 أمتار خطّ نظر؛ تختزله الجدران وأجهزة الميكروويف كثيرًا.
