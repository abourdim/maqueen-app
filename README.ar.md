# Maqueen Lab

**مختبر مكوّنات BLE على الويب لروبوت DFRobot Maqueen Lite v4 + الأطقم الميكانيكيّة.**

كلّ مُحرِّك ومستشعر على Maqueen Lite متاح من بطاقة في تبويب **🤖 Maqueen** — قُد العجلات، حرِّك السيرفوهات، أضِئ الـLEDs والـNeoPixels، شغِّل الجرس، قِس بالموجات فوق الصوتيّة، اقرأ جهاز التحكّم بالأشعة تحت الحمراء، اتبع خطًّا. تبويب ثانٍ **🧪 Playground** يحتفظ بالتبويبات الفرعيّة القديمة لـbit-playground (Controls، Sensors، Graph، 3D، Bench، More) للتجارب الحرّة مع micro:bit. كلّ التحكّم يجري عبر BLE UART ؛ لكلّ أمر رقم تسلسلي وتأكيد بالصدى.

كذلك يحوي المستودع :

- **`labs/`** — 8 **مختبرات** لطيفة للأطفال، كلّ منها مركَّز على قدرة واحدة (عصا تحكّم، مسافة، موسيقى، سيرفوهات، IR، إضاءة، رؤية، مساعد قيادة). كلّ مختبر يثبِّت على اليمين **سجلّ رسائل** أمينًا لتنسيق TX/RX في التطبيق الرئيسي، والمظاهر/اللغات تطابق سطح الوثائق (`carbon / forest / steel / paper / pearl`).
- **`docs/`** — دليل المستخدم، خريطة الأطراف، الخطّة، المخطَّطات (متغيِّرات FR / EN / AR لكلٍّ منها).
- **`workshops/`** — موادّ A4 قابلة للطباعة بجودة PDF : **دليل** ثنائي اللغة، **كتيِّب**، **بطاقات سريعة**، **منشِّطات**، **مركز**، إضافة إلى **منشور** و**ملصق** جذّابَين للأطفال (FR، 8 سنوات+، انفجارات قصصيّة، QR حقيقيّة → التطبيق المباشر، تحجيم تلقائي على الجوّال).

> **الحالة :** v0.1.60 — تبويب Maqueen مكتمل ؛ سطح Labs (8 مختبرات) منشور ؛ أزرار FAB قابلة للسحب في الكوكبيت ؛ تحديث العلامة `ROBI-9 LAB → MAQUEEN LAB` ؛ منشور وملصق v2 مع QR. راجع [CHANGELOG.md](docs/CHANGELOG.md) للتطوّرات الأخيرة، و[plan.md](docs/plan.md) لخطّة البناء الأصليّة.

## العتاد

DFRobot Maqueen Lite v4 (ROB0148) — روبوت تعليمي مبنيّ على micro:bit. أطقم ميكانيكيّة اختياريّة : Forklift (ROB0156-F)، Loader (ROB0156-L)، Beetle gripper (ROB0156-B)، Push (ROB0156-P).

### خريطة الأطراف (مرجع سريع)

| الطرف | يستخدمه | ملاحظات |
|---|---|---|
| P0 | الجرس | طرف `music` الافتراضي |
| P1 | الموجات فوق الصوتيّة TRIG | تعارض : منفذ Gravity |
| P2 | الموجات فوق الصوتيّة ECHO | تعارض : منفذ Gravity |
| P8 | LED اليسرى (بسيط) | رقمي ON/OFF |
| P12 | LED اليمنى (بسيط) | رقمي ON/OFF |
| P13 | مستشعر الخطّ الأيسر | رقمي 0/1 |
| P14 | مستشعر الخطّ الأيمن | رقمي 0/1 |
| P19/P20 | ناقل I2C | محرّك + RGB + سيرفوهات على العنوان **0x10** |

### خريطة سجلّات I2C 0x10

| السجلّ | الوظيفة |
|---|---|
| `0x00` | المحرّك M1 (يسار) |
| `0x02` | المحرّك M2 (يمين) |
| `0x14` | السيرفو S1 |
| `0x15` | السيرفو S2 |
| `0x32` | بيانات 4× LEDs RGB |

خريطة الأطراف الكاملة وتحذيرات التعارض ومواصفات البروتوكول في [docs/USER_GUIDE.md](docs/USER_GUIDE.md).

## بداية سريعة

1. اِفلش `firmware/v1-maqueen-lib.ts` على micro:bit (الصق في [makecode.microbit.org](https://makecode.microbit.org)، أضف امتداد `pxt-maqueen`، نزِّل ملفّ `.hex`).
2. شغِّل Maqueen Lite.
3. افتح تطبيق الويب في Chrome / Edge (Web Bluetooth مطلوب).
4. اضغط **Connect**، اقترِن مع `BBC micro:bit [xxxxx]`.
5. اختر مستكشف مكوّن والعب.

## التشغيل محلّيًا

**باختصار — لا تنقر مرّتين على `index.html`.** قواعد CORS في Chromium تمنع
`fetch()` لـ`manifest.json` / `product.json` / `build-info.json` من
بروتوكول `file://`. التطبيق يتدهور بلطف (لا أثر وظيفي) لكن
وحدة تحكّم DevTools تمتلئ بالأخطاء الحمراء، ولا يظهر طلب تثبيت PWA،
ولا يستطيع Service Worker التسجيل.

اختر المُشغِّل الذي بحوزتك — كلّها تخدم `http://localhost:8000` :

| المنصّة | الأمر | ملاحظات |
|---|---|---|
| **Windows** | نقرة مزدوجة `serve.bat` | يجرِّب Python، ثمّ `npx serve` |
| **macOS / Linux** | `./serve.sh` | يجرِّب Python، ثمّ `npx serve` |
| **أيّ نظام مع Python 3.7+** | `python tools/serve.py` | المكتبة القياسيّة فقط، بدون تثبيت |
| **أيّ نظام مع Node** | `npm run serve` | يستخدم `npx serve` (one-shot من CDN) |
| **Python عبر npm** | `npm run serve:py` | يشغِّل `tools/serve.py` |

مُشغِّل Python يقبل منفذًا مخصَّصًا (`python tools/serve.py 8765`)
ويفتح المتصفِّح الافتراضي إلّا إذا ضُبط `MAQUEEN_NO_BROWSER=1`.

## البنية

- **تطبيق الويب** : JS + SVG + CSS عاديّون. لا إطار. قابل للتثبيت كـPWA.
- **BLE** : Web Bluetooth → خدمة Bluetooth UART لـmicro:bit. مُسلسِل عالميّ واحد للكتابة ينتظر كلّ وعد `writeValue()` — لا مزيد من `NetworkError: GATT operation already in progress`. حالة الاتّصال تُبثّ عبر إشارة DOM + MutationObserver ؛ الإرسالات المعلّقة تُرفض عند الانفصال.
- **البروتوكول** : كلّ أمر يحمل رقم تسلسل. يردّ micro:bit بـ`ECHO:N <verb>`. يعرض تطبيق الويب المُرسَل / المُصدى / المفقود / متوسّط زمن الاستجابة على رقاقة BLE bench في شريط المستشعرات. `HELLO`/`HELLO:<ver>` يبلِّغ نسخة البرنامج الثابت على بطاقة Connect.
- **تدفّقات المستشعرات** : ACC / LIGHT / SOUND تتدفّق تلقائيًّا مع نبضة قلب (≥ مرّة كلّ ~500–1000 ميلي ثانية) كي لا يبدو Graph مجمَّدًا. مغلقة افتراضيًّا — اقلب رقاقة `streams: ON/OFF` في الشريط، أو ادخل تبويب Sensors / Graph / 3D وستتسلَّح تلقائيًّا. DIST / LINE / IR تُسبَر عند الطلب مع شرائح معدّل سبر (200–2000 مللي ثانية، محفوظة في localStorage). تتوقَّف الموسوبات تلقائيًّا عند مغادرة تبويب Maqueen لتوفير عرض BLE.
- **البرنامج الثابت** : يبقى غبيًّا. تطبيق الويب يقود كلّ الرسوم المتحرّكة (مسح، وميض، قوس قزح، تكّة متابعة الخطّ) عبر أوامر متدفّقة.
- **مرآة USB التسلسليّة** بسرعة 115200 baud — لافتة الإقلاع، سجلّ RX/TX، الأوامر المنفَّذة. مفيدة للتنقيح بدون تطبيق الويب.

### نسخة البناء / البرنامج الثابت

خطّاف pre-commit يرفع `BUILD_VERSION` تلقائيًّا **فقط** عندما يكون `firmware/v1-maqueen-lib.ts` ضمن التغيير المهيَّأ. لم تعد commits الوثائق وحدها ترفع النسخة — صار الختم يتتبَّع تغييرات البرنامج الثابت الحقيقيّة. ملفّ `.hex` **لا** يُترجَم تلقائيًّا ؛ أعِد بناءه في MakeCode وأعِد الفلش عند تغيير النسخة (انظر قسم "Building the firmware .hex" في دليل المستخدم).

راجع [plan.md](docs/plan.md) للخطّة الكاملة.
